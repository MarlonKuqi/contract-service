chore: release version 2.0.0

BREAKING CHANGES:
- PagedContractResponse: renamed 'first' → 'isFirst', 'last' → 'isLast'

Features:
- Contract startDate defaults to now() when not provided (as per specs)
- ContractPeriod validates endDate > startDate (throws 422 if invalid)
- Boolean fields prefixed with 'is' (Vaudoise guidelines compliance)

Improvements:
- Refined HTTP status codes for better REST conformity:
  * 422 for all data/validation errors (simple & consistent)
  * 400 for missing query parameters
  * 404 for resources not found
  * 409 for conflicts (duplicate email/identifier)
  * 403 for forbidden access (contract ownership)
- Added HttpMessageNotReadableException handlers in both ClientControllerAdvice
  and ContractControllerAdvice for consistency
- Removed handler from GlobalExceptionHandler (proper separation of concerns)

Tests:
- Updated 8 pagination test assertions for isFirst/isLast
- Validated all HTTP status codes (400, 404, 409, 422, 403, 500)
- All integration tests passing (80%+ coverage)

- Proof API works section with:
